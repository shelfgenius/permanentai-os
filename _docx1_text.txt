3.2 Data Flow â€” Step by Step
Step 1 â€” Site Input
User enters address or drops pin. OpenLayers resolves lat/lng. Geo engine queries: seismic zone (P100-1/2013), snow load zone (CR 1-1-3), wind pressure zone (CR 1-1-4), climate zone (energy), municipal zoning (POT/CUT/H limits).
Step 2 â€” Mode Selection
ARCHITECTURAL mode focuses on spatial layout, facades, circulation, furniture, finishes, lighting design, accessibility (HG 923/2010). ENGINEERING mode focuses on structural grid, load paths, FEA, section sizing, foundation design, MEP routing, energy compliance.
Step 3 â€” Model Building
User sketches or imports geometry. FreeCAD/Bonsai handles parametric BIM authoring. IfcOpenShell assigns IFC4 types: IfcWall, IfcBeam, IfcColumn, IfcSlab, IfcDoor, etc. AutoCAD layer standard applied automatically (A-WALL, S-BEAM, E-POWER, P-WATER...).
Step 4 â€” AI Agent Debate
Each specialist agent analyses the model and raises issues or proposals. E.g. Structural Agent flags column spacing > 6m without intermediate support. Architectural Agent objects to a load-bearing wall conflicting with open-plan intent. Debate Moderator arbitrates, proposes compromise (e.g. concealed steel beam). All agent reasoning is logged and shown in the UI.
Step 5 â€” Calculation Pass 1 (Primary Engine)
OpenSeesPy runs FEA: load combinations per SR EN 1990 (ULS/SLS), seismic analysis per EC8/P100 (response spectrum or time-history), beam/column/slab section design per EC2/EC3. OpenStudio runs energy simulation. Results stored in calculation database.
Step 6 â€” Safety Layer (Pass 2)
INDEPENDENT engines re-run identical problem: AnaStruct for 2D frames, PyNite for 3D frames, SciPy direct stiffness for simple checks. Results compared numerically. Divergence > 2% triggers a RED FLAG â€” user is warned, calculation is blocked until resolved.
Step 7 â€” Code Compliance Check
Automated rule engine checks every result against: P100 drift limits, EC2 reinforcement ratios, EC3 slenderness limits, energy U-values (Legea 372), fire resistance ratings (P118), accessibility (HG 923), zoning POT/CUT/height limits. Pass/Fail shown per item.
Step 8 â€” Output Generation
Blueprint DXF generated with ezdxf (correct AutoCAD layers, scales, title block). PDF generated with ReportLab (ISO A0â€“A4, SR ISO 5457 title block). IFC4 exported for use in Revit, Archicad, BIMserver. Calculation report PDF with full derivations, assumptions, code references.

5.2 Debate Protocol
Trigger:  Any design decision that affects two or more agent domains triggers a debate session.
Round 1 â€” Positions:  Each affected agent states its position and the specific code clause that supports it.
Round 2 â€” Objections:  Agents raise objections to conflicting positions with quantified impact (cost/safety/performance delta).
Round 3 â€” Proposals:  Each agent proposes a compromise that satisfies its hard constraints while accommodating others.
Moderation:  Debate Moderator scores proposals. Hard code violations = disqualified. Remaining proposals ranked by multi-criteria score. Top proposal presented to user.
User Override:  User may override the recommendation. System logs the override, notes the code implication, and requires explicit acknowledgement.

6. SAFETY LAYER â€” DUAL-PASS CALCULATION VERIFICATION
PRINCIPLE: No structural, energy, or compliance result is accepted until it has been independently re-calculated by a second engine. Divergence above tolerance automatically blocks output and escalates to the engineer. This mirrors the peer-checking obligation in real engineering practice (four-eyes principle).
6.1 Architecture of the Safety Layer
The Safety Layer sits between the Calculation Engine and the Output Generator. It is architecturally isolated â€” it has read-only access to inputs and cannot be influenced by Pass 1 results before running its own calculation.
PASS 1 â€” Primary Engine
OpenSeesPy (structural FEA) + OpenStudio (energy) + IfcOpenShell-derived geometry quantities
PASS 2 â€” Verification Engine
AnaStruct (2D frames), PyNite (3D frames), SciPy direct stiffness method (simple cases), hand-formula checks for standard beam/column cases
Comparator
Numerical diff of: reactions, member forces, moments, deflections, story drifts, energy loads, reinforcement areas
Tolerance Bands

Override Policy
A licensed engineer may override a YELLOW flag with signature. RED flags cannot be overridden â€” the model must be corrected.
Audit Log
Every calculation, both passes, tolerance result, and any overrides are stored in immutable audit log (timestamped, hashed).
